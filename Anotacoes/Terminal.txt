====================
ESTRUTURA FHS LSB ->
====================

------

* LSB = ou (Linux Standard Base) é uma especificação que define padrões para sistemas operacionais Linux
* FHS = File System Hierarchy Standard, VEIO DA ESPECIFICAÇÃO LSB, Padrão de Hierarquia de Sistema de Arquivos

------

DIRETÓRIOS DO FHS -->

Significados dos diretórios (extraido de https://pt.wikipedia.org/wiki/Filesystem_Hierarchy_Standard):

/bin - Binários de usuários, essenciais no boot
/sbin - Binários do superusuário, essenciais no boot
/boot - Arquivo do gerenciador de partida e kernel, símbolos
/dev - Dispositivos do sistema
/etc - Arquivos de configuração globais
 /etc/opt - Arquivos de configuração para aplicativos em /opt
 /etc/X11 - Arquivos de configuração para o X Window System 11
/home - Armazenamento de dados de contas de usuários normais
/root - Armazenamento de dados de contas do superusuário
/lib - Bibliotecas essenciais do sistema, de binários localizados em /bin e /sbin
/mnt - Sistema de arquivos montado temporariamente
/media - Ponto de montagem de mídias removíveis (como pen-drives, cd-rom)
/opt - Pacotes estático de aplicações
/proc - systema de arquivos virtual, onde pode fazer a interação com o kernel
     e processos do sistema
/tmp - Arquivos temporários. Conteúdo geralmente apagado no reboot nas distribuições
/usr - (unix system resources) - Hierarquia secundária (não essenciais no boot) para dados compartilhados de usuários
  /usr/bin - O mesmo que a hierarquia /bin, mas contém binários não essenciais ao
  funcionamento da máquina ou para o recovery
  /usr/include - Diretório padrãod para headers
  /usr/lib - O mesmo que a hierarquia /lib, mas não essenciais ao boot
  /usr/sbin - O emsmo que o /sbin, mas não essenciais ao boot da máquina
  /usr/share - Dados compartilhados independentes de arquitetura
  /usr/src - Armazenamento de código fonte da máquina
  /usr/X11R6 - - X Window Sysem, versão 11R6
  /usr/local - Armazenamento de binários não distribuidos na instalação principal
         da máquina, ou seja, fora do sistema de empacotamento. Também é o local
         de armazenamento terciário de dados
/var - Arquivos que são gravados comf requencia (logs, páginas web, email, imagens, etc)
  /var/lock - Arquivos de lock, usados para controlar corretamente os recursos em uso
  /var/log - Arquivos de log, usado para logs em geral
  /var/mail - Caixas de e-mail dos usuários do sistema em formato mailbox
  /var/run - Contém dados sobre a execução do sistema desde seu primeiro boot 
          (daemons e usuários)
  /var/spool - Spooling de tarefas (fila de impressão, cache de pacotes, proxy, etc)
  /var/spool/mail - Antigo local da caixa de correio de usuários (deve ser usado /var/mail)
  /var/tmp - Arquivos temporários. Quando usado em modo multi-usuário.
  
MINHAS ANOTAÇÕES ---->

* /BIN = BINÁRIOS DO SISTEMA, SÃO ARQUIVOS EXECUTÁVEIS DO SISTEMAS ex = comando (ls) (vim) (cd)

* /BOOT = VAI TER ARQUIVOS ESSENCIAIS PARA A INICIALIZAÇÃO DO SISTEMA COMO POR EXEMPLO O (GRUP)
(GRUP INICIALIZADOS DO SISTEMA) TEMOS TAMBÉM O (vmlinuz, é o kernel do linux) 

* /DEV = DEVICE, VAI SER AONDE VAI FICAR MEUS DISPOSITIVOS EX-> (meu HD QUE É O SDA)

* /ETC = DIRETÓRIO AONDE VAI TER OS ARQUIVOS DE CONFIGURAÇÃO DA MINHA MÁQUINA COMO POR EX-> (hostname), temos configurações de redes também como (network), temos o  (motd, ou mensagem do dia)

* /HOME = AONDE VAI FICAR OS DIRETÓRIOS HOME DO USUÁRIO

* initrd.img = IMAGEM DE INICIALIZAÇÃO

* /LIB = AONDE VÃO FICAR AS BIBLIOTECAS como por ex-> dlls no windao, vamos ter (modules, versão do kernel (.ko) são os drivers )

* /lib32, /libx32, lib64 = SÃO AONDE VÃO TER BIBLIOTECAS E MÓDULOS DO KERNEL LINUX

* LSOT+FOUND = SÃO SISTEMAS DE ARQUIVOS PERDIDOS, ACABOU A ENERGIA ELE CONSEGUE RECUPERAR ESSES ARQUIVOS CORROMPIDOS

* /MEDIA = QUALQUER COISA QUE EU ESPETAR NA MÁQUINA POR DEFAULT ELE VAI SER MONTADO NO /MEDIA

* /MNT = PONTO DE MONTAGEM TEMPORÁRIO, EX-> MONTAR ARQUIVO ISO 

* /OPT = PONTO AONDE EU DEVERIA INSTALAR UM SOFTWARE

* /PROC = FILESYSTEM VIRTUAL, VOU TER INFORMAÇÕES DINÂMICAS SOBRE O SISTEMA E PROCESSOS EX-> (cpu info)

* /ROOT = ÚNICO DIRETÓRIO HOME DO USUÁRIO ROOT FORA DO /HOME

* /RUN = VAI ME TRAZER INFORMAÇÕES SOBRE TUDO OQ ESTA RODANDO DÊS DO ULTIMO BOOT, ELE É CONSIDERADO UM ARQUIVO TEMPORÁRIO (TMPFS) DADOS DINÂMICOS, TODA VÊS QUE EU REBOOT A MÁQUINA AS INFO SOME 

* /SBIN = BINÁRIOS EXÊNCIAIS DO SUPER USER, COMANDOS COMO (ADICIONAR USUÁRIO, DELETAR USUÁRIO, MUDAR O USER, PWUCONV,)

* /SRV = SÃO DADOS NÃO DINÂMICOS, MAS SIM ESTÁTICOS, SERVIÇOS DE VERSIONAMENTO, PÁGINAS DO MEU WEB SERVE, DADOS SERVIDOS PELO SISTEMA

* /SYS = INFORMAÇÕES REFERENTES AO SISTEMA, RELACIONADO AO KERNEL, DEVICES, DRIVERS, PARECIDO COM O RUN MAS SÓ QUE ELE VAI ME TRAZER INFORMAÇÕES DO DEVICES

* /TMP = USADOS PARA COLOCAR DADOS TEMPORÁRIOS PELAS APLICAÇÕES

* /USR = BINÁRIOS DE USO EXCLUSIVO DO ROOT SEGUNDA ESTRUTURA DE DIRETÓRIOS DO LINUX, AQUI VAMOS TER COMANDOS NÃO EXÊNCIAIS PARA O SISTEMA, VAMOS TER (LIB, BIBLIOTÊCAS DINÂMICAS PARA OS BIN E SBIN DENTRO DO USR)
	/LOCAL = VAMOS TER UMA OUTRA ESTRUTURA PARECIDA, TERCEIRA HIERARQUIA DE DIRETÓRIOS

* /VAR = CONTÉM INFORMAÇÕES SOBRE VARIÁVEIS COMO POR EXEMPLO (LOG), aonde terá informações do log como kernel debug etc, (BACKUPS, CACHE, LOCAL, MAIL) ETC....
